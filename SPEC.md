# AB Songbook — Техническое задание

> Персональное веб-приложение для гитариста  
> Версия 1.0 · Февраль 2026

---

## 1. Описание проекта

AB Songbook — персональное веб-приложение для хранения, организации и использования текстов песен с гитарными аккордами. Приложение заточено под два основных сценария: разучивание новых песен дома и живая игра в компании друзей.

### Проблемы, которые решает

- Песни разбросаны по разным сайтам, десятки открытых вкладок в браузере
- Аккорды есть только в первом куплете — неудобно для новых песен
- Тональность сбивается при каждом посещении сайта
- Невозможно добиться компактного размещения текста на экране телефона
- Слишком большой выбор на сайтах отвлекает от практики
- Нет способа быстро поделиться песней с друзьями в живой ситуации

### Целевая аудитория

Гитаристы-любители, играющие на акустической гитаре, поющие под аккомпанемент. Уровень: от начинающего до среднего. Типичный контекст — домашние репетиции и дружеские посиделки.

## 2. Сценарии использования

### Сценарий А: Разучивание дома

- Устройства: компьютер, планшет
- Открывает приложение, находит песню или добавляет новую
- Вводит текст с аккордами в формате ChordPro
- Подстраивает тональность под свой голос, запоминает
- Использует автопрокрутку для игры с экрана
- Повторяет, пока песня не выучена

### Сценарий Б: Игра в компании

- Устройства: телефон (основной), редко планшет
- Шум, суета, плохое освещение, нет времени листать
- Нужен максимально крупный текст, в идеале вся песня на одном экране
- Быстрое переключение между песнями из подготовленного сет-листа
- Друзья сканируют QR-код или получают ссылку, чтобы видеть текст и подпевать
- Опция: показывать или скрывать аккорды для подпевающих / второй гитары

## 3. Функциональные требования

### 3.1. Хранение песен

Внутренний формат — ChordPro. Каждая песня содержит: название, исполнитель, оригинальная тональность, теги, текст с аккордами, индивидуальные настройки (транспонирование, размер шрифта). Данные сохраняются между сессиями.

### 3.2. Список песен (главный экран)

- Полнотекстовый поиск по названию и исполнителю
- Теги/категории: «для компании», «разучиваю», «рок», «бардовская» и т.д.
- Фильтрация по тегам, сортировка
- Счётчик песен

### 3.3. Просмотр песни (режим игры)

Главный экран приложения — здесь пользователь проводит 90% времени.

- Аккорды на отдельной строке над текстом (классический формат песенника)
- Аккорды во всех куплетах, а не только в первом
- Транспонирование ±полутон, настройка запоминается для каждой песни
- Регулировка размера шрифта текста и аккордов отдельно
- Настройка цвета аккордов (6 вариантов)
- Опция моноширинного шрифта для точного выравнивания аккордов
- Автоподстройка под экран — вся песня на одном экране, быстрое вкл/выкл
- Автопрокрутка с регулируемой скоростью
- Визуальное выделение припева и бриджа (нейтральный серый фон + боковая линия)
- Переключение между песнями из текущего сет-листа

### 3.4. Редактор песни

- Поля: название, исполнитель, тональность, теги
- Текстовое поле для ввода в формате ChordPro
- Живой предпросмотр
- Поддержка секций: куплет, припев, бридж

### 3.5. Сет-листы

- Именованные подборки песен: «у Серёги на даче», «новогодняя»
- Своя очерёдность песен
- Навигация между песнями: кнопки «пред./след.»

### 3.6. Шаринг

Три способа поделиться:

- QR-код — показать экран, друг сканирует камерой
- Ссылка/код для отправки в мессенджер
- Копирование текста песни в буфер обмена для вставки в чат

Настройки шаринга:

- Чекбокс «показывать аккорды» — выкл для подпевающих, вкл для второй гитары
- Можно поделиться одной песней или всем списком

### 3.7. Импорт / Экспорт

- Импорт из файла ChordPro (.cho, .chordpro, .txt)
- Импорт из вставленного текста
- Экспорт всей коллекции в ChordPro-файл (бэкап)
- Экспорт в PDF (планируется)

### 3.8. Персонализация

- Светлая и тёмная тема
- Цвет аккордов — 6 предустановленных вариантов с превью
- Размер аккордов относительно текста (от −3 до +3)
- Опция моноширинного шрифта для текста песни

## 4. Нефункциональные требования

- Работает в браузере на телефоне, планшете, компьютере без установки
- Быстрый запуск, нет задержек при переключении между песнями
- Офлайн-работа (после первой загрузки, при реализации как PWA)
- Данные сохраняются между сессиями
- Компактный интерфейс, минимум лишних элементов
- Приватность: шаринговые ссылки могут сгорать через заданное время (планируется)

## 6. Техническая архитектура

### Текущий стек (прототип)

- Frontend: React (один файл JSX, автономный компонент)
- Стилизация: inline styles с динамической темизацией
- Хранение данных: persistent storage API (key-value)
- Шаринг: shared storage для обмена через коды доступа
- Рендеринг аккордов: собственный парсер ChordPro → React-компоненты
- Иконки: встроенные SVG (без зависимостей)
- QR-код: SVG-генератор (для MVP, заменить на библиотеку qrcode)

### Целевой стек (продакшен)

- Frontend: React + Vite
- Backend: Node.js сервер
- Хранение: серверная БД (SQLite или PostgreSQL)
- Шаринг: серверные ссылки с TTL (время жизни)
- PWA: Service Worker для офлайн-работы
- QR: библиотека qrcode для настоящих сканируемых QR-кодов

### Формат данных

Песни хранятся в формате ChordPro — открытый текстовый стандарт. Пример:

```
{title: Звезда по имени Солнце}
{artist: Кино}
{key: Am}

{sov}
[Am]Белый снег, [C]серый лёд
[D]На растрескавшейся [F]земле
{eov}

{soc: Припев}
[Am]Две тысячи лет — [F]война
[Am]Война без особых [E]причин
{eoc}
```

Директивы секций: `{sov}/{eov}` — куплет, `{soc: Припев}/{eoc}` — припев, `{sob}/{eob}` — бридж.

## 8. План развития

### Фаза 1: Перенос и стабилизация

- Перенести проект в Claude Code (React + Vite + Node.js)
- Подключить серверную БД вместо browser storage
- Заменить генератор QR на библиотеку qrcode (сканируемые коды)
- Реализовать шаринг по URL (серверные ссылки)

### Фаза 2: Удобство и надёжность

- PWA-манифест и Service Worker для офлайн-работы
- Экспорт в PDF
- TTL для шаринговых ссылок (приватность)
- Drag-and-drop сортировка песен в сет-листе
- Дублирование песни (быстрое создание варианта)

### Фаза 3: Расширение функционала

- Диаграммы аккордов (аппликатуры) для новых аккордов
- Импорт с популярных сайтов (парсинг, если формат позволяет)
- Синхронизация экранов в реальном времени (WebSocket)
- Встроенный метроном

## 9. Глоссарий

| Термин | Описание |
|--------|----------|
| ChordPro | Открытый текстовый формат для записи песен с аккордами. Аккорды в [квадратных скобках] перед словом. |
| Транспонирование | Смещение всех аккордов на заданное количество полутонов вверх или вниз для подстройки под голос. |
| Сет-лист | Упорядоченный список песен для конкретного выступления или мероприятия. |
| PWA | Progressive Web App — веб-приложение, которое можно установить на телефон и использовать офлайн. |
| TTL | Time To Live — время жизни ссылки, после которого она перестаёт работать. |
| MVP | Minimum Viable Product — минимально рабочая версия продукта. |
