# AB Songbook — Техническое задание

> Персональное веб-приложение для гитариста  
> Версия 2.0 · Апрель 2026

---

## 1. Описание проекта

AB Songbook — персональное веб-приложение для хранения, организации и использования текстов песен с гитарными аккордами. Приложение заточено под два основных сценария: разучивание новых песен дома и живая игра в компании друзей.

### Проблемы, которые решает

- Песни разбросаны по разным сайтам, десятки открытых вкладок в браузере
- Аккорды есть только в первом куплете — неудобно для новых песен
- Тональность сбивается при каждом посещении сайта
- Невозможно добиться компактного размещения текста на экране телефона
- Слишком большой выбор на сайтах отвлекает от практики
- Нет способа быстро поделиться песней с друзьями в живой ситуации

### Целевая аудитория

Гитаристы-любители, играющие на акустической гитаре, поющие под аккомпанемент. Уровень: от начинающего до среднего. Типичный контекст — домашние репетиции и дружеские посиделки.

## 2. Сценарии использования

### Сценарий А: Разучивание дома

- Устройства: компьютер, планшет
- Открывает приложение, находит песню через sidebar (поиск, исполнитель, статус)
- Вводит текст с аккордами в формате ChordPro
- Подстраивает тональность через per-song controls в sidebar
- Использует автопрокрутку для игры с экрана
- Меняет статус песни: Новая → Учу → Знаю

### Сценарий Б: Игра в компании

- Устройства: телефон (основной), редко планшет
- Шум, суета, плохое освещение, нет времени листать
- Нужен максимально крупный текст, в идеале вся песня на одном экране
- Быстрое переключение между песнями из подготовленного сет-листа
- Друзья получают ссылку, чтобы видеть текст и подпевать

## 3. Функциональные требования

### 3.1. Хранение песен

Внутренний формат — ChordPro. Каждая песня содержит: название, исполнитель, оригинальная тональность, статус (new/learning/known), текст с аккордами, индивидуальные настройки (транспонирование, размер шрифта). Данные в PostgreSQL.

### 3.2. Sidebar (боковая панель)

Центр управления приложением:
- Поиск по названию/исполнителю
- Фильтр по статусу (Все/Новые/Учу/Знаю)
- Список исполнителей (быстрый фильтр)
- Сет-листы (навигация + редактирование)
- Admin actions (добавить, импорт, экспорт)
- Настройки: темы, 8 цветов с hex-вводом, просмотр
- Per-song controls при просмотре песни

Сворачивается в полоску с иконками (48px). На мобильных — drawer.

### 3.3. Таблица песен (главный экран)

- Столбцы: Песня, Исполнитель, Тональность, Статус, Дата добавления
- Сортировка по клику на заголовок, sticky header
- Быстрое переключение статуса кликом
- Меню действий (три точки): редактировать, добавить в сет-лист
- Фильтры отображаются как chips с кнопкой удаления

### 3.4. Просмотр песни (режим игры)

Главный экран приложения — здесь пользователь проводит 90% времени.

- Аккорды на отдельной строке над текстом
- Аккорды во всех куплетах
- Транспонирование через sidebar (per-song)
- Регулировка шрифта и интервала через sidebar
- Настройка цветов через sidebar (8 цветов)
- Опция моноширинного шрифта
- Автоподстройка под экран (fit-to-screen)
- Автопрокрутка с регулируемой скоростью
- Визуальное выделение припева и бриджа
- Кнопка "Редактировать" под заголовком (admin)

### 3.5. Редактор песни

- Поля: название, исполнитель, тональность, статус
- Текстовое поле ChordPro на всю высоту экрана
- Живой предпросмотр (переключатель)
- Кнопка "Сохранить" рядом с формой (не в header)

### 3.6. Сет-листы

- Именованные подборки песен
- Навигация prev/next с фиксированной шириной заголовка
- Редактирование: таблица с чекбоксами
- Добавление песни в сет-лист из меню трёх точек

### 3.7. Персонализация

- 4 темы-пресета (тёмная, светлая, контрастная, тёплая)
- Полный редактор цветов: 8 цветов (текст, muted, аккорды, фон, surface, border, припев, бридж)
- Hex-ввод + color picker
- Кастомизации сохраняются per-theme, переключение тем не теряет изменения
- Кнопка "Сбросить цвета темы"
- Размер аккордов, моноширинный шрифт, нотация B/H

### 3.8. Импорт / Экспорт

- Импорт из файла ChordPro (.cho, .chordpro, .txt)
- Импорт из вставленного текста
- Экспорт всей коллекции в ChordPro-файл

## 4. Нефункциональные требования

- Работает в браузере на телефоне, планшете, компьютере без установки
- Быстрый запуск, нет задержек при переключении между песнями
- Данные сохраняются между сессиями
- Управление преимущественно мышкой (sidebar, клики)
- Принцип UI: элементы управления рядом с контентом, не в дальних углах

## 5. Техническая архитектура

### Стек

- Frontend: React + Vite + Tailwind CSS v4
- Backend: Node.js + Express
- БД: PostgreSQL 15
- Контейнеры: Docker Compose (postgres, server, client)
- Reverse proxy: nginx + certbot SSL
- Домен: DuckDNS

### Формат данных

Песни хранятся в формате ChordPro. Пример:

```
{title: Звезда по имени Солнце}
{artist: Кино}
{key: Am}

{sov}
[Am]Белый снег, [C]серый лёд
[D]На растрескавшейся [F]земле
{eov}

{soc: Припев}
[Am]Две тысячи лет — [F]война
[Am]Война без особых [E]причин
{eoc}
```

## 6. План развития

### Ближайшие задачи

- Mobile responsive (drawer sidebar, адаптация таблицы)
- Многоколоночный текст песни (1-2-3-4 столбца для десктопа)
- Фон/рамка аккордов (настраиваемый стиль)

### Следующий этап

- Аппликатуры аккордов с вариантами (панель справа)
- Мини-плеер YouTube
- Версии сложности аккордов (оригинал/упрощённые)

### Далее

- Автоподбор тональности по диапазону голоса
- Подбор аккордов по YouTube/mp3
- PWA / офлайн
- Экспорт в PDF

## 7. Глоссарий

| Термин | Описание |
|--------|----------|
| ChordPro | Открытый текстовый формат для записи песен с аккордами. Аккорды в [квадратных скобках] перед словом. |
| Транспонирование | Смещение всех аккордов на заданное количество полутонов вверх или вниз. |
| Сет-лист | Упорядоченный список песен для конкретного мероприятия. |
| Sidebar | Боковая панель с навигацией, фильтрами и настройками. |
| Per-song controls | Настройки привязанные к конкретной песне (транспонирование, шрифт, масштаб). |
