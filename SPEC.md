# AB Songbook — Техническое задание

> Персональное веб-приложение для гитариста  
> Версия 2.1 · Апрель 2026

---

## 1. Описание проекта

AB Songbook — персональное веб-приложение для хранения, организации и использования текстов песен с гитарными аккордами. Приложение заточено под два основных сценария: разучивание новых песен дома и живая игра в компании друзей.

### Проблемы, которые решает

- Песни разбросаны по разным сайтам, десятки открытых вкладок в браузере
- Аккорды есть только в первом куплете — неудобно для новых песен
- Тональность сбивается при каждом посещении сайта
- Настройки не синхронизируются между устройствами
- Невозможно добиться компактного размещения текста на экране телефона
- Слишком большой выбор на сайтах отвлекает от практики

---

## 2. Сценарии использования

### Сценарий А: Разучивание дома

- Устройства: компьютер, планшет
- Открывает приложение, находит песню через sidebar (поиск, исполнитель, статус)
- Подстраивает тональность — настройка сохраняется на сервере, доступна на всех устройствах
- Использует автопрокрутку для игры с экрана
- Меняет статус песни: Новая → Учу → Знаю

### Сценарий Б: Игра в компании

- Устройства: телефон (основной), редко планшет
- Быстрое переключение между песнями из сет-листа (галерейная навигация)
- Масштаб подстраивается под экран устройства автоматически
- Все темы и настройки уже синхронизированы с домашнего компьютера

## 3. Функциональные требования

### 3.1. Хранение песен

Формат — ChordPro. Каждая песня: название, исполнитель, оригинальная тональность, статус (new/learning/known), транспонирование, текст с аккордами. Данные в PostgreSQL.

### 3.2. Sidebar (боковая панель)

Центр управления:
- Поиск по названию/исполнителю
- Фильтр по статусу (Все/Новые/Учу/Знаю)
- Список исполнителей (быстрый фильтр)
- Сет-листы (активный выделен, навигация + редактирование)
- Admin actions (добавить, импорт, экспорт)
- Настройки: темы, 8 цветов с hex-вводом, интервал
- Per-song controls при просмотре песни (транспонирование, масштаб, автопрокрутка)

Сворачивается в полоску с иконками (48px). На мобильных — drawer.

### 3.3. Таблица песен (главный экран)

- Столбцы: Песня, Исполнитель, Тональность, Статус, Дата добавления
- Сортировка по клику на заголовок, sticky header
- Быстрое переключение статуса кликом (публичный API, без admin)
- Единое меню [⋮]: редактировать, статус, сет-листы, удалить
- Фильтры как chips с кнопкой удаления

### 3.4. Просмотр песни (единый SongView)

Один компонент для просмотра из любого контекста (список, сет-лист).

- Заголовок в одну строку: `Исполнитель — Название [⋮]`
- Кликабельный исполнитель → фильтр
- Единое меню [⋮] — то же что в таблице
- Галерейная навигация: hover по краям текста → полупрозрачные стрелки prev/next
- Навигация через текущий фильтр или сет-лист (navList)
- Шрифт: только моноширинный (точное позиционирование аккордов)
- Размер аккордов = размер текста (не настраивается отдельно)
- Автопрокрутка с регулируемой скоростью
- Fit-to-screen (per-device)
- Визуальное выделение припева и бриджа

### 3.5. Редактор песни

- Поля: название, исполнитель, тональность, статус
- Текстовое поле ChordPro на всю высоту
- Живой предпросмотр (переключатель)
- Кнопки рядом с формой (не в header)

### 3.6. Сет-листы

- Именованные подборки песен
- Навигация через SongView (галерейные стрелки или swipe)
- Активный сет-лист выделен в sidebar
- Меню [⋮] рядом с каждым сет-листом в sidebar (редактировать, удалить)
- Редактор: компактные строки (название+исполнитель в одной ячейке) с чекбоксами + стрелки ↑↓ для порядка + три точки
- Фильтр в редакторе: все песни / только в сет-листе
- Добавление песни в сет-лист из меню [⋮] на странице песни или в таблице

### 3.7. Настройки и синхронизация

**На сервере (PostgreSQL, таблица settings):**
- 4 темы-пресета с пользовательскими модификациями (customThemes, 12 цветов)
- Размер шрифта, интервал
- showChords, useH (нотация B/H), chordStyle (нет/фон/рамка/оба)

**Per-song в БД (таблица songs):**
- Транспонирование (songs.transpose) — глобальный атрибут песни

**Per-device (localStorage):**
- Текущая тема (каждое устройство выбирает свою)
- fitScale, columns — зависят от экрана устройства
- navList (sessionStorage) — сохраняется при F5
- Состояние sidebar (open/closed)

### 3.8. Стиль аккордов

- Переключатель: нет → фон → рамка → фон+рамка (цикл)
- Отдельный цвет шрифта аккордов (`chords`) и фона (`chordBg`)
- Цвет акцентов UI (`accent`) отделён от цвета аккордов
- Позиционирование не нарушается (моноширинный шрифт)

### 3.8. Импорт / Экспорт

- Импорт из файла ChordPro (.cho, .chordpro, .txt)
- Импорт из вставленного текста
- Экспорт всей коллекции в ChordPro-файл

## 4. Нефункциональные требования

- Работает в браузере на телефоне, планшете, компьютере без установки
- Быстрый запуск, нет задержек при переключении между песнями
- Настройки синхронизируются между устройствами через сервер
- На desktop управление мышкой, на touch — swipe для навигации
- Принцип UI: элементы управления рядом с контентом, не в дальних углах
- Все интерактивные элементы с cursor-pointer и hover-эффектом

## 5. Mobile / touch-устройства

- Sidebar — drawer overlay (полностью скрыт когда закрыт)
- Плавающая кнопка-hamburger в левом верхнем углу
- Swipe left/right по тексту песни — переход к prev/next
- Gallery-arrows скрыты на touch (JS-детект через navigator.maxTouchPoints)
- Word-wrap: пары "аккорд+текст" переносятся как единицы, аккорд всегда над словом
- Интервал (lineHeight) — per-song per-device для гибкой подстройки под экран

## 6. Правая панель

Симметрично левому sidebar — 280px справа на desktop (настраивается 200-500px), drawer на mobile. Открывается кнопкой-grid в заголовке SongView.

**Аппликатуры аккордов (секция "Аккорды"):**
- Уникальные аккорды из текущей песни (с учётом transpose + нотации B/H)
- Библиотека `@tombatossals/react-chords` + `chords-db` (~500 аккордов)
- Grid с auto-fit по выбранному размеру диаграмм (70-250px, +/- в UI)
- Клик по диаграмме — цикл по вариантам позиций (индикатор X/N)
- Номер лада слева от диаграммы на уровне barre (SVG viewBox 80×70 с translate(13,13))
- Нормализация: если baseFret=1 но все frets ≥ 2 без открытых струн — сдвигаем для правильного отображения лада
- Редкие аккорды (не в базе) — placeholder "не найден"

**YouTube плеер (секция "YouTube"):**
- До 3 ссылок с кастомными названиями (`youtube_urls[]` + `youtube_labels[]`)
- Табы переключения с пользовательскими лейблами ("Оригинал", "Разбор", "Кавер" и т.д.)
- iframe 16:9

**Ресайз ширины** мышкой через draggable border, сохранение в localStorage per-device.
**Размер аппликатур** — кнопки +/- в секции, per-device localStorage.
